What is docker?

Docker is best described as an application which runs other applications. Docker 'containerises" applications such as web servers, databses, and more in to an easy to run and downloadble package. With docker, near instantaneuosly, you can spin up as many LAMP servers or Scale an application as much as one may want. While docker is used heavily in the computing world and in the industry, docker can also be used for some home use. This tutorial will teach you how to install docker and how to run your first docker image on your machine.

How to setup Docker on the Raspberry Pi

  1. To start, Start by SSHing into your raspberry pi. This can be done by typing "ssh -l root (your ip address)". Once you hit enter, the raspberry pi will then prompt you for your password.
  2. Now that we are connected to the Pi, we are going to want to type "dietpi-software". This will launch us into the installation page for different software packages.
  3. Once you are in the software menu, select the "Software Optimised" tab and press enter. You are going to want to scroll down until you find the package called Docker. Press spacebar on this and then press enter to install it.
  4. After this, tab down to the Install button and hit enter. Give it some time while everything downloads. Once everything is downloaded, the raspberry pi will prompt you to restart, go ahead and press ok.
  5. Once the Pi restarts reconnect via SSH again. Docker should now be running and enabled on your pi. To test this, type "docker -v" to see what version of docker you are running.
  6. Now that we have confirmed that docker is installed we can run our first docker instance. To start, you are going to want to type:

    git clone https://github.com/dockersamples/node-bulletin-board

    This will clone the sample docker file from dockers Github Repo.
  7. Next, type:

    cd node-bulletin-board/bulletin-board-app

    This will navigate to the files folder we pulled from the docker github.
  8. Once in this folder, we need to type:

    docker build --tag bulletinboard:1.0 .

    this will build our docker image.
  9. Finally, to run our rocker image we must type

    docker run --publish 8000:8000 --detach --name bb bulletinboard:1.0

    To break this down command down some, "docker run" tells us t we are going to run a docker image, "--publish 8000:8000" says that we are publishin on board 8000 and we open that port on our device, "--detach" tells our application to run in the background, and "--name" names the process running in the background.
  10. Now that our docker image is running, we can navigate to it by going to yourip:8000 in a web browser of your choice. If all went to plan, then you should see a website for a bulletin board!
